## Task execution order

Tasks execute sequentially through the ADM lifecycle. Each task receives context from its predecessors.

## Phase 1: Engagement Strategy & Delivery Planning

| Property | Value |
|----------|-------|
| Agent | Associate Director |
| Context | None (first task) |
| Output file | `deliverables/01_engagement_strategy.md` |

**Description:** Define the overall engagement strategy and delivery approach. Clarify business objectives, success metrics, delivery model, governance structure, and high-level risks.

**Expected output:** Business objectives and success criteria, recommended delivery model, high-level roadmap, governance model, key risks and mitigations.

---

## Phase 2: Strategic Engagement Approval

| Property | Value |
|----------|-------|
| Agent | Senior Delivery Lead |
| Context | Phase 1 |
| Output file | `deliverables/02_strategic_approval.md` |

**Description:** Review and approve the engagement strategy. Validate commercial viability and executive stakeholder alignment. Authorize proceeding to the Analyze phase.

**Expected output:** Go/no-go recommendation, strategic alignment assessment, resource and timeline validation, authorization to proceed.

---

## Phase 3: Business Requirements Analysis

| Property | Value |
|----------|-------|
| Agent | Strategy & Consulting Analyst |
| Context | Phase 2 |
| Output file | `deliverables/03_business_requirements.md` |

**Description:** Analyze client needs and translate them into structured functional and non-functional requirements.

**Expected output:** Business problems, prioritized functional requirements, non-functional requirements, assumptions and risks.

---

## Phase 4: Backlog Definition & Scope Planning

| Property | Value |
|----------|-------|
| Agent | Management Consultant |
| Context | Phase 3 |
| Output file | `deliverables/04_backlog_scope.md` |

**Description:** Translate requirements into a prioritized backlog with clear scope, acceptance criteria, and sprint planning.

**Expected output:** Epics and user stories, acceptance criteria, scope boundaries, dependencies, effort estimates.

---

## Phase 5: Solution Architecture Design

| Property | Value |
|----------|-------|
| Agent | Solution Architect |
| Context | Phase 4 |
| Output file | `deliverables/05_solution_architecture.md` |

**Description:** Design the end-to-end solution architecture addressing scalability, security, integration, and operability.

**Expected output:** Architecture overview, components and integrations, data flows, architectural decisions, technology stack.

---

## Phase 6: Technical Architecture Review

| Property | Value |
|----------|-------|
| Agent | Technology Architect |
| Context | Phase 5 |
| Output file | `deliverables/06_technical_architecture_review.md` |

**Description:** Validate the solution architecture from a technical perspective. Ensure enterprise-grade standards, security compliance, and scalability.

**Expected output:** Validation report, technical risk assessment, compliance review, performance analysis, approval recommendation.

---

## Phase 7: Solution Development & Implementation

| Property | Value |

## Phase 12: Service Introduction & Engagement Closure

|----------|-------|
