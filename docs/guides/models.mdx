## Default model

The crew defaults to **Hermes 4 405B** by NousResearch — a reasoning model optimized for agentic tool-use workflows. All agents (including the manager) use the same model.

## Switching models

### In code

Use the `set_model()` method before calling `.crew()`:

```python
from accenture_adm_hierarchical_delivery_crew.crew import (
    AccentureAdmHierarchicalDeliveryCrew,
)

crew = AccentureAdmHierarchicalDeliveryCrew().set_model("DeepSeek V3.2").crew()
crew.kickoff(inputs={"client_name": "Acme", "engagement_type": "Cloud Migration"})
```

### Adding a new model

Add an entry to `AVAILABLE_MODELS` in `crew.py`:

```python
AVAILABLE_MODELS = {
    "Hermes 4 405B": "openai/NousResearch/Hermes-4-405B",
    "Your New Model": "openai/vendor/model-name",
    # ...
}
```

<Warning>
  All model IDs must be prefixed with `openai/` for litellm routing. The bare model ID (e.g., `deepseek-ai/DeepSeek-V3.2`) won't work.
</Warning>

## Model compatibility notes

- **CrewAI's hierarchical mode** constructs multi-turn conversations with interleaved system messages. Some models (e.g., Qwen) reject this with a "System message must be at the beginning" error. DeepSeek V3.2 and Hermes 4 405B handle it correctly.
- **Reasoning models** (Hermes, DeepSeek R1, GPT-OSS) tend to produce more structured, thorough deliverables but cost more per token.
- **Chat models** (DeepSeek V3.2, Gemma, Llama) are faster and cheaper but may produce less detailed analysis.
